# 🎮 bmiibo - 虛擬玩具生態系 (Virtual Toy System)

`bmiibo` 是一個開源、跨平台且完全免費的**虛擬玩具檔案格式與開源生態系**。它的核心理念與實體 amiibo 類似，但完全數位化、輕量化。

我們致力於為**獨立遊戲、Minecraft 模組、Scratch/TurboWarp 創作者以及教育程式平台**帶來全新的「數位收藏儀式感」與安全、防刷的防禦性載入機制。

---

## 📂 生態系大本營目錄結構

本儲存庫採用 Monorepo（單一儲存庫）架構管理，您可以根據需求進入各平台的專屬領域：

* **[`/python`](./python)**：官方 Python 核心套件。支援自訂異常攔截、5MB 安全安檢，並內建 Pygame 紋理轉換支援。已正式發布至 PyPI（`pip install bmiibo`）。
* **[`/turbowarp`](./turbowarp)**：TurboWarp (Scratch) 專用外掛積木。採用無沙盒模式與內建 JSZip，具備本機每日限讀一次的防刷限制。
* **[`/manager_app`](./manager_app)**：專為玩家打造的 **Tkinter 數位公仔牆展示櫃**。一橫排 4 隻網格排版，自動清點與統計收藏總數，極具收藏儀式感。
* *🚀 預留未來擴充：[`/minecraft`](#) (Fabric / NeoForge 模組 API 專區)*

## 📦 檔案格式規範 (.bmiibo)

`.bmiibo` 本質上是一個**自訂副檔名的壓縮檔（ZIP 格式）**。為了確保各平台（尤其是網頁端的 TurboWarp）能夠以極速解開並讀取，所有標準 `.bmiibo` 檔案**必須採用 `ZIP_STORED`（僅儲存、不壓縮）**模式進行封裝。

內部包含以下標準結構：

```text
my_character.bmiibo
 ├── bmiibo.json      # 🧠 核心數據（名稱、唯一 ID、創作者、各平台專屬數值）
 └── icon.png         # 🎨 虛擬玩具的視覺圖示（建議標準尺寸 512x512 PNG）
📄 標準 bmiibo.json 範本

{
    "id": "custom_steve_001",
    "name": "我的特製史蒂夫",
    "bmiibo_version": "1.0",
    "author": "Huang Zi-ji",
    "description": "這是一隻具有強大防禦屬性的特製虛擬公仔。",
    "game_data": {
        "turbowarp": {
            "score_multiplier": 2
        },
        "pygame": {
            "bonus_hp": 50,
            "attack_power": 15
        }
    }
}
🔒 核心三大安全防線
不論在何種平台載入 bmiibo，官方核心皆實作了以下防禦性軟體工程規範：

5MB 嚴格防崩潰限制：拒絕載入任何大於 5MB 的偽裝檔案，防止惡意超大檔案撐爆遊戲記憶體。

每日限讀一次（冷卻防刷）：系統會在本地產生冷卻記帳本。同一隻玩具 ID 在 24 小時之內拒絕重複刷卡發放獎勵，完美複製實體公仔的冷卻體驗。

格式完整性校正：嚴格校驗 ZIP 內部是否完整包含 bmiibo.json 與 icon.png，格式損壞時會拋出明確的自訂異常（Exceptions），確保遊戲主程式絕對不閃退。
